## 1. Auto-update 체크

- [ ] 1.1 세션 시작 시 버전 확인 경로를 default-off로 연결한다.
- [ ] 1.2 네트워크 실패를 세션 실패와 분리하고 warn-only로 처리한다.
- [ ] 1.3 쿨다운 / TTL 상태를 저장하는 최소 구조를 만든다.

## 2. 운영 알림

- [ ] 2.1 새 버전 발견 시 경고만 남기고 자동 업데이트는 하지 않는다.
- [ ] 2.2 반복 알림이 TTL 안에서 중복되지 않는지 확인한다.
- [ ] 2.3 current version과 latest version이 같을 때는 무반응이어야 한다.

## 3. 검증

- [ ] 3.1 default-off / enabled / failure 경로를 각각 테스트한다.
- [ ] 3.2 실제 레지스트리 조회 1회와 모킹 테스트를 분리해 검증한다.
- [ ] 3.3 릴리스 기능이 core harness 흐름을 막지 않는지 확인한다.
